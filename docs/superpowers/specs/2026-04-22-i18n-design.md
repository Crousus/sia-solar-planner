# i18n Design — English + German

- **Date:** 2026-04-22
- **Status:** Draft (pending user review)

## Motivation

Solar Planner's UI is entirely in English. Adding German (DE) support opens the app to German-speaking solar installers — the primary target market — without requiring a full localisation platform. Two languages with a manual toggle plus browser auto-detection is the right scope for now.

## Scope

**Translated:** all functional UI text — field labels, button text, placeholder text, error messages, status strings, section headers, empty states, modal copy.

**Not translated:** decorative brand/tech-label strings (e.g. `FIG_00 · ACCESS TERMINAL`, `SOLAR / PLANNER · v0.1`, `↵ RETURN TO SUBMIT`, tool keyboard shortcut hints). These are intentional English-language design elements and remain untouched.

## Dependencies

Three npm packages added to `dependencies`:

| Package | Purpose |
|---|---|
| `i18next` | Core translation engine |
| `react-i18next` | React bindings (`useTranslation` hook) |
| `i18next-browser-languagedetector` | Auto-detects browser language; reads/writes `localStorage` |

## Architecture

### Translation files

```
src/locales/en.ts   — English strings, `as const` object (type source-of-truth)
src/locales/de.ts   — German strings, typed `satisfies typeof en` (missing keys = compile error)
```

Using `.ts` (not `.json`) so TypeScript enforces completeness at build time — adding a key to `en.ts` without a matching key in `de.ts` is a compile error.

### Initialisation

`src/i18n.ts` — configures and exports the i18next instance. Imported once as a side-effect in `main.tsx` before the React tree mounts.

Configuration:
- `LanguageDetector` plugin enabled
- Detection order: `localStorage` → `navigator.language` → fallback `en`
- Cache: write detected/chosen language back to `localStorage` key `i18nextLng`
- `fallbackLng: 'en'`
- Resources: both `en` and `de` loaded eagerly (no lazy-loading needed for two languages)

### Usage in components

```tsx
const { t } = useTranslation();
// ...
<span>{t('login.signIn')}</span>
```

`t()` is fully typed via the `i18next` TypeScript plugin — only valid keys compile.

### Language toggle

A small `EN / DE` pill added to the right side of the `Toolbar`, adjacent to `SyncStatusIndicator`. Clicking the inactive language calls `i18n.changeLanguage(lang)`, which triggers a re-render and writes the choice to `localStorage` automatically via the detector plugin. The active language segment is visually distinguished (underline or slightly brighter).

## Key namespace structure

Single flat resource bundle per language, dot-notation namespaced by feature area:

| Namespace | Components |
|---|---|
| `login.*` | `LoginPage` |
| `toolbar.*` | `Toolbar` |
| `sidebar.*` | `Sidebar` |
| `team.*` | `TeamPicker`, `TeamView`, `TeamMembers`, `NewTeamPage` |
| `conflict.*` | `ConflictModal` |
| `sync.*` | `SyncStatusIndicator` |

## Components touched

| Component | What gets translated |
|---|---|
| `LoginPage` | Field labels, button text (`Sign in`, `Create account`), mode toggle text, error messages, placeholder text |
| `NewTeamPage` | Form label, placeholder, button text |
| `TeamPicker` | Empty-state text, button label |
| `TeamView` | Section headers, empty states, project action labels |
| `TeamMembers` | Invite field label/placeholder, status chip labels, button text |
| `ConflictModal` | Modal title, body copy, action button labels |
| `Toolbar` | Tool mode labels (`Roof`, `Panels`, `String`, `Delete`), action buttons (`Lock Map`, `Export`, `Save`, `Load`, `Reset`), basemap option labels |
| `Sidebar` | All panel section headers, field labels, stat labels, helper text |
| `SyncStatusIndicator` | Status strings (`Saved`, `Saving…`, `Offline`, `Error`) |

## Language toggle placement

The toggle lives in the `Toolbar` right section, before `SyncStatusIndicator`. It is a minimal inline control — two text segments separated by `/`, active one distinguished visually. It does not use a dropdown to keep the footprint small and avoid adding interaction weight to an already-dense toolbar.

## What is not in scope

- More than two languages
- Server-side language preference (stays in `localStorage` only)
- Number/date/currency formatting (`Intl` primitives) — no locale-sensitive numbers in the current UI
- Plural forms — current strings have no pluralisation needs
- Translation management tooling (Lokalise, Weblate, etc.)
